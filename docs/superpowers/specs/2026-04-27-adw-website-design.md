# ADW Website (v1) вЂ” Design Spec

## 1. Context

Adelaide Design Week (ADW) needs a public-facing website to support the 2026
program. The existing `apenning-hub/everywhere-ghost` repo is a Ghost theme
(Handlebars + Ghost backend) and is heavier than what's needed right now. This
spec covers a new, separate, statically-hosted site that reuses ADW's existing
visual identity (yellow `#FEFF35`, Univers LT Std Bold + Egizio Regular, the
asterisk motif, lowercase headings, Kaurna country acknowledgment) but drops the
Ghost runtime, membership, and email features for now.

EOIs for ADW 2026 open **Thursday 29 April 2026** вЂ” two days from this spec's
date вЂ” so v1 must ship fast. v1 is intentionally minimal and content-driven;
content lives in markdown so it can be edited without touching templates.

## 2. Goals

- Public, statically-hosted multi-page site for ADW 2026.
- Authoring in markdown вЂ” non-technical edits to copy don't require touching
  HTML or templates.
- Reuse ADW's existing visual identity: yellow palette, the two custom fonts,
  lowercase headings, asterisk motif, footer Kaurna acknowledgment.
- Deploy to **both** GitHub Pages (mirror) and Cloudflare Pages (custom domain
  owner) from the same repo, same build.
- EOI page is the priority surface вЂ” it carries the FAQ, key dates, eligibility,
  a form preview, and a CTA out to the existing Tally form.

## 3. Non-goals (v1)

- No membership/email subscriptions (no Ghost members, no mailing list signup).
- No on-site EOI form вЂ” submissions live entirely in Tally; the site only links
  out.
- No CMS вЂ” content is markdown in the repo, edited via PR or direct push.
- No blog/news feed; no post listings, tagging, pagination, author pages.
- No analytics, no third-party scripts, no embeds beyond static images.
- No real per-year pages under "Past ADW" вЂ” just an index with year stubs.

## 4. Architecture

**Static site generated by Eleventy (11ty).** Markdown source files become HTML
at build time. One build artifact (`_site/`) is deployed to two hosts.

**Why 11ty over alternatives:**
- Jekyll: GH Pages auto-builds it, but Cloudflare Pages still needs a build
  step, and Ruby tooling is heavier locally for someone in a Node ecosystem.
- Plain HTML + client-rendered markdown: bad first paint, bad SEO, fragile
  without JS вЂ” wrong for a public site.
- 11ty: single `package.json`, fast builds, native markdown, layouts/partials
  let the 7 pages share one header/footer, both target hosts deploy it
  natively.

## 6. Visual identity & CSS strategy

**Trim, don't rewrite.** Source: `assets/css/screen.css` from
`apenning-hub/everywhere-ghost` (~1440 lines).

**Keep, verbatim:**
- `@font-face` blocks for Univers LT Std Bold and Egizio Regular.
- `:root` custom-property block: yellow `#FEFF35`, greys, font stacks, spacing
  scale, shadow tokens.
- Reset, base typography (lowercase headings via `text-transform: lowercase` on
  `h1вЂ“h6`, `clamp()`-based responsive sizing), `body { background: var(--yellow) }`.
- Footer: preserved verbatim from the Ghost theme вЂ” `logo-no-tagline.svg`, the
  tagline row ("adelaide design week" / "thinking. designing. making."), the
  nav row (links + contact + Instagram), the Kaurna country acknowledgment
  block, and the asterisk SVG icon.

**Drop:**
- `.post-card*`, `.posts-grid`, `.pagination*`, `.newsletter-*`, `.member-*`,
  modal/menu animations specific to the old home, `.index-*` post-listing
  styles, anything tied to Ghost membership.

**Add (small, new):**
- `.site-nav` вЂ” top nav with 7 tab links, active-state underline, sensible
  mobile collapse (wraps to two rows; no JS hamburger needed for v1).
- `.nav-tile` вЂ” the four big tiles on Home (About / EOI / Program / Past ADW),
  bold and large, using the existing palette (e.g. yellow-on-grey or
  grey-on-yellow flip on hover).
- `.page` / `.prose` вЂ” readable column for markdown body content (max-width
  ~720вЂ“880px, generous line-height, the existing typographic scale).
- `.faq` вЂ” uses native `<details>/<summary>` for FAQ accordions (no JS).
- `.cta-tally` вЂ” the EOI submit button, prominent, reusing existing button
  vocabulary.
- `.key-dates` вЂ” boxed/bordered scannable dates block on the EOI page.
- `.disclaimer` вЂ” small, bordered, muted block at bottom of EOI page.

**Typography rules:**
- Site headings (`h1вЂ“h4`) stay lowercase вЂ” part of the identity.
- FAQ `<summary>` text stays in sentence case (the questions are real questions
  and read naturally that way).

**Net result:** `site.css` of roughly 300вЂ“500 lines vs the source 1440.

## 8. FAQ content (EOI page)

Rendered as `<details>/<summary>` per question, no JS required. The 13 Q&As:

1. Who can apply? вЂ” Anyone working in design or related fieldsвЂ”individuals,
   studios, collectives, venues, organisations.
2. Do I need to be based in Adelaide? вЂ” No. But your project must take place
   in, and have a clear connection to Adelaide.
3. What can I submit? вЂ” Projects (exhibitions, talks, installations, etc.),
   offers to collaborate, venues, team involvement, or sponsorship.
4. Do I need a venue? вЂ” No. You can request help with venue matching, however;
   projects with confirmed venues may be prioritised.
5. Can I submit more than one EOI? вЂ” Yes. Submit each one separately.
6. What stage does my project / idea need to be at? вЂ” Any stageвЂ”concept to
   complete. It must be viable, and realistically and professionally
   deliverable by the program dates (14вЂ“18 October).
7. How are projects selected? вЂ” Based on idea, relevance, feasibility, and how
   it contributes to the overall program. Assessed according to the T&C
   criteria.
8. What support do you provide? вЂ” Program inclusion (listing), promotion, and
   as needed - connections (venues, collaborators, etc.). Limited production
   or funding support available at the discretion of ADW team.
9. Is funding provided? вЂ” Generally; no. This is a platform, not a funding
   body.
10. What am I responsible for? вЂ” Delivering your project, insurance,
    install/deinstall, and being available before, during, and after the event
    for contact and communications with the ADW team.
11. Can I charge for events? вЂ” Yes. You will be in charge of managing your own
    sales and systems if you choose to do this.
12. Does submitting an EOI guarantee a place in the program? вЂ” No. All EOIs are
    reviewed and assessed according to the T&C's.
13. Can I update my proposal later? вЂ” Minor changes are fine. Major changes
    need approval.

## 9. Deployment

**Build pipeline (shared by both hosts):**
- Local dev: `npm run dev` в†’ 11ty serves on `localhost:8080` with hot reload.
- Build: `npm run build` в†’ outputs static site to `_site/`.
- Node version pinned to 20 (in `package.json` `engines` and CI/Cloudflare
  config).

**GitHub Pages:**
- `.github/workflows/deploy.yml` runs on push to `main`: `actions/checkout` в†’
  `actions/setup-node@v4` в†’ `npm ci && npm run build` в†’ `actions/upload-pages-artifact@v3`
  в†’ `actions/deploy-pages@v4`.
- Repo Settings в†’ Pages в†’ Source = **GitHub Actions** (no `gh-pages` branch).
- `src/CNAME` is passthrough-copied into `_site/` so GH Pages picks it up.
- Live at `apenning-hub.github.io/adw-website` as a mirror.

**Cloudflare Pages:**
- Connect the same GitHub repo in the Cloudflare dashboard.
- Build command: `npm run build`. Output dir: `_site`. Node version: 20.
- Auto-deploys on push, independent of the GH Actions workflow.
- Custom domain attached in Cloudflare Pages в†’ Custom domains. **Cloudflare
  owns the apex.** GH Pages serves only its `*.github.io` URL.

**Why both:** Cloudflare Pages serves the live custom domain; GH Pages is a
no-cost always-on mirror at the `*.github.io` URL. Both build from the same
`main` branch, so they stay in sync.

## 12. Risks / things to verify

- **Font licensing.** Univers LT Std Bold and Egizio Regular are bundled in the
  source repo as `.otf` files. Confirm ADW has the right to redistribute these
  fonts on a public static site. If not, swap to comparable open alternatives
  before launch.
- **Two hosts on the same repo.** Both GH Actions and Cloudflare Pages will
  fire on every push to `main`. That's fine but doubles CI minutes вЂ” accept,
  not a blocker.
- **Tally URL stripping.** The provided URL had Facebook tracking params;
  spec uses the clean canonical (`https://tally.so/r/5BllMv`). Verify this
  resolves to the same form.
- **Time pressure.** EOIs open 29 April. Implementation plan should optimise
  for shipping the EOI page first; other pages can land as stubs.

## 13. Out of scope (defer to v2 or later)

- Email/newsletter subscription.
- On-site EOI form (Tally remains the source of truth).
- Per-year pages under Past ADW with full content.
- Search, filtering, tags.
